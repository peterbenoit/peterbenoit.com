Form Fields

How to use

  • There are 3 sizes of form fields - regular/medium/small - regular is the default size in all instances, medium/small should be used on smaller screens where the regular size is too big
  • Form fields are for users to input information into
  • States include: default, disabled, error, warning, active, focus
  • Errors are triggered to match corresponding requirement feedback messages
  • Warnings are triggered to match corresponding recommendation feedback messages
  • Active states are triggered when a user clicks into a field
  • Focus states are triggered when a user uses a keyboard to tab through fields
  • It should be easy to troubleshoot any errors – i.e. it should be clear what information is missing through feedback messages
  • Consider using icons to enforce specific actions - form field icons should be on the right (Ex: search fields)
