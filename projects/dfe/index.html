Introduction

A component library is a collection of pre-designed, reusable UI components that can be used to create a consistent and efficient user interface. By using a component library, developers can save time and effort as they don't have to design and build individual components from scratch every time they need to create a new feature or page. This leads to more efficient development and can also improve the overall user experience by providing a cohesive visual design.

What is a design system?

  • A design system is a tool for organizing information and standards about product UIs

  • Design systems provide

    • Interactive demos of UI components that are considered standards for a website or digital product

    • Design principles, guidance, and best practices to support user experiences, accessibility, and mobile-friendly content display and interactions

  • Design system components may include

    • Specific elements like buttons and typography

    • Patterns like page layouts and how combinations of elements should work together

    • Specifications describing the usage and behaviors

    • Visual design details

  • Design systems may sometimes contain code snippets that development teams can use

Why do we need a design system?

  • Serves as an important communication tool

    • Within design teams

    • To/from design and development teams

    • To/from stakeholders

  • Supports consistency within a product UI

  • Provides transparency - design and development team is considering previously established UI standards

  • Becomes the record or source of approved or final designs

  • Helps ID gaps and stay organized throughout UI enhancements

Who uses a design system?

  • UX/UI Designers

  • UI Developers

  • Business Analysts

  • QA Testers

  • Product Owners/Managers

  • Product Stakeholders

What will the DFE Design System (DDS) include?

  • Global styling elements such as typography and colors

  • Component guidance for usage and best practices

  • Component design specifications

  • Screen layout specifications

  • How components fit together to build screens

  • Notes on screen sizes and grid systems

How can you view the DDS?

  • Establish a centralized domain for DDM-lead design systems - designsystems.cdc.gov

  • This site would only be available on the CDC network

  • Anticipate at least 2 high priority design systems:

  • DFE (https://designsystems.cdc.gov/dfe)

  • TP5 (https://designsystems.cdc.gov/tp5)

  • Others as needed (e.g. web apps needing to align to TP5)

  • Environments mirror those we use for regular site publishing

  • https://designsystems-dev.cdc.gov work in progress, UX team still defining design elements/patterns, demos/examples/specs still in flux

  • https://designsystems-test.cdc.gov design elements ready for review by the internal team

  • https://designsystems-stage.cdc.gov prep for production (reminder this would not be going to a public-facing site)

  • https://designsystems.cdc.gov what we consider design standards for use in DFE and TP5 UI dev, what QA and stakeholders would reference most of the time
